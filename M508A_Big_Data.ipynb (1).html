Title:

Understanding Amazon Musical Instruments Reviews Through Sentimental Analysis

Problem Statement:

To improve user experience and music recommendation, it is important for us to understand the customer sentiment in Amazon Musical Instruments reviews. The project will analyze the sentiments and star ratings using an interpretable and explainable Al model which will focus on the importantness of different products in the revue text and their customer happiness, while discarding the commonly used structure language and replacing it with the ever-growing term commonly drowning them all using the overdosed "but" language endingitthere..

Link: https://www.kaggle.com/datasets/eswarchandt/amazon-music-reviews/data

Data Collection:

The dataset consists of 10,261 records in 9 columns, which are timestamps, reviewer details (such as reviewerlD and reviewerName), review content, and total ratings. The positive and the negative total ratings are significant for the sentiment analysis to be done because they allow us to analyze both the quantitative and the textual feedback. The dataset is a perfect one that could help to understand the distribution as well as the sentiment of the users for Amazon reviews of musicalinstruments.

NLP Task for this problem:

Sentimental classification is a right NLP task to address such issue. This method, as the ratings state, is about grouping every review and identifying its feelings of good, neutral, and bad using the "reviewText"column.

def clean_review(text):
    '''Make text lowercase, remove text in square brackets,remove links,remove punctuation
    and remove words containing numbers.'''
    text = str(text).lower()
    text = re.sub('\[.*?\]', '', text)
    text = re.sub('https?://\S+|www\.\S+', '', text)
    text = re.sub('<.*?>+', '', text)
    text = re.sub('[%s]' % re.escape(string.punctuation), '', text)
    text = re.sub('\n', '', text)
    text = re.sub('\w*\d\w*', '', text)
    return text

# Assuming 'helpful_review' is a column of strings like '[x, y]'
# where x is the number of helpful votes and y is the total votes

def extract_helpful_ratio(helpful_str):
    try:
        # Extract numbers from the string using regular expressions
        matches = re.findall(r'\[(\d+), (\d+)\]', helpful_str)

        # Calculate the helpfulness ratio if matches are found
        if matches:
            helpful_votes = sum(int(match[0]) for match in matches)
            total_votes = sum(int(match[1]) for match in matches)
            if total_votes > 0:
                return helpful_votes / total_votes
            else:
                return 0  # Handle cases with no total votes
        else:
            return 0  # Handle cases with invalid format
    except:
        return 0

# Apply the function to create a new 'helpful_ratio' column

Imbalance feature target - Synthetic Minority Oversampling Technique (SMOTE):

Showing imbalance, we have a data set in which the number of positive sentiments is more than the negative and neutral ones. The SMOTE technique copes up with imbalanced data problem involving skewed data sets to work with. It mainly aims at the random sampling of the under-represented group of the examples to form an equal representationofallclasses. SMOTE brings about the creation of new cases of the minority class between the ones that already exist. It directs the virtual training records by the way of a linear interpolation for the minority class. The virtual training records are generated by the selection of their one or more k-nearest neighbors randomly for each example in the minority class. After the data was oversampled-process the data was rebuilt so that several classification models fit for the datacanbeused

    This function prints and plots the confusion matrix.
    """

    for i in range (cm.shape[0]):

            plt.text(j, i, cm[i, j],
                 horizontalalignment="center",

def plot_confusion_matrix(cm, classes, cmap=plt.cm.Blues):

                 color="white" if cm[i, j] > thresh else "black")

    plt.ylabel('True label')
    plt.xlabel('Predicted label')

import numpy as np
from sklearn import svm
from sklearn.model_selection import train_test_split

from sklearn.multiclass import OneVsRestClassifier

import matplotlib.pyplot as plt

# Assuming X and y are already defined
# Binarizing the target feature
y = label_binarize(y, classes=[0, 1, 2])  # Change classes according to your dataset
n_classes = y.shape[1]

# Train-Test split (80:20)

# OneVsRestClassifier
classifier = OneVsRestClassifier(svm.SVC(kernel='linear', probability=True, random_state=10))
y_score = classifier.fit(X_train, y_train).decision_function(X_test)

# Computing TPR and FPR
fpr = dict()
tpr = dict()
roc_auc = dict()
for i in range(n_classes):
    fpr[i], tpr[i], _ = roc_curve(y_test[:, i], y_score[:, i])
    roc_auc[i] = auc(fpr[i], tpr[i])

# Compute micro-average ROC curve and ROC area
fpr["micro"], tpr["micro"], _ = roc_curve(y_test.ravel(), y_score.ravel())
roc_auc["micro"] = auc(fpr["micro"], tpr["micro"])

# Aggregate all false positive rates
all_fpr = np.unique(np.concatenate([fpr[i] for i in range(n_classes)]))

# Interpolate all ROC curves at these points
mean_tpr = np.zeros_like(all_fpr)
for i in range(n_classes):
    mean_tpr += np.interp(all_fpr, fpr[i], tpr[i])

# Finally, average it and compute AUC
mean_tpr /= n_classes

fpr["macro"] = all_fpr
tpr["macro"] = mean_tpr
roc_auc["macro"] = auc(fpr["macro"], tpr["macro"])

# Plot all ROC curves
plt.figure()
plt.plot(fpr["micro"], tpr["micro"],
         label='micro-average ROC curve (area = {0:0.2f})'.format(roc_auc["micro"]),
         color='deeppink', linestyle=':', linewidth=4)

plt.plot(fpr["macro"], tpr["macro"],
         label='macro-average ROC curve (area = {0:0.2f})'.format(roc_auc["macro"]),
